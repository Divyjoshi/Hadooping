What is Hadoop?

Hadoop is an open-source framework that allows for the distributed processing of large data sets across clusters of computers using simple programming models. It is designed to scale up from single servers to thousands of machines, each offering local computation and storage.

Before we begin the actual installation process, ensure your machine meets the following requirements:

  • Java Development Kit (JDK): Hadoop is written in Java, so you‚Äôll need the JDK installed on your system. You can download it from Oracle's JDK Download Page.

  • A Linux environment: Hadoop runs on Linux, so you'll need to have it installed. If you're on Windows, consider using WSL (Windows Subsystem for Linux).

Let's Go !

Step 1: Update Your System

Before installing Hadoop, make sure your system is up-to-date with the latest packages.

sudo apt update && sudo apt upgrade

Step 2: Install Java Development Kit (JDK)

Hadoop is written in Java, so the first step is to install the Java Development Kit. Here's how you can do that:

sudo apt update && sudo apt install openjdk-8-jdk

Step 3: Verify Java Installation

After installation, it‚Äôs important to confirm that Java is correctly installed. Check your Java version with the following command:

java -version

Step 4: Install SSH

SSH, or Secure Shell, is used to securely connect to different nodes of the Hadoop cluster. Install it with this command:

sudo apt install ssh

Step 5: Create a Hadoop User

Create a separate user account for Hadoop to improve security and manage permissions easily:

sudo adduser hadoop

Step 6: Switch to Hadoop User

After creating the Hadoop user, switch to it:

su - hadoop

Step 7: Set Up SSH for Hadoop User

AFor Hadoop to function properly, it needs to communicate between its different nodes without being prompted for a password. Set this up by generating an SSH key and adding it to the list of authorized keys:

ssh-keygen -t rsa
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 640 ~/.ssh/authorized_keys

Step 8: Test SSH

Ensure SSH is set up correctly by connecting to your localhost:

ssh localhost

Step 9: Install Hadoop

Now, let‚Äôs get Hadoop itself:

wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz

Unpack the downloaded file:

tar -xvzf hadoop-3.3.6.tar.gz

And rename the folder for convenience (optional):

mv hadoop-3.3.6 hadoop

Step 10: Configure Environment Variables

Set up the necessary environment variables by editing the ~/.bashrc file:

nano ~/.bashrc

Append the following lines to the file:

export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export HADOOP_HOME=/home/hadoop/hadoop
export PATH=$PATH:$HADOOP_HOME/sbin:$HADOOP_HOME/bin

Additionally, set JAVA_HOME in the hadoop-env.sh file to point to your Java installation.

nano $HADOOP_HOME/etc/hadoop/hadoop-env.sh

Step 11: Configure Hadoop

Create directories for the namenode and datanode:

cd hadoop
mkdir -p ~/hadoopdata/hdfs/{namenode,datanode}

Edit the core-site.xml file to include your system's hostname and configure the default filesystem parameter.

nano $HADOOP HOME/etc/hadoop/core-site.XML

Copy the below code.

<configuration>
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://localhost:9000</value>
</property>
</configuration>

Proceed to configure the hdfs-site.xml by setting the paths for NameNode and DataNode.

nano $HADOOP HOME/etc/hadoop/hdfs-site.XML

Copy the below code.

<configuration>
<property>
    <name>dfs.replication</name>
    <value>1</value>
</property>
<property>
    <name>dfs.namenode.name.dir</name>
    <value>file:///home/hadoop/hadoopdata/hdfs/namenode</value>
</property>
<property>
    <name>dfs.datanode.data.dir</name>
    <value>file:///home/hadoop/hadoopdata/hdfs/datanode</value>
</property>
</configuration>

Update the mapred-site.xml with the correct environmental variables.

nano $HADOOP HOME/etc/hadoop/mapred-site.XML

Copy the below code

<configuration>
<property>
    <name>yarn.app.mapreduce.am.env</name>
    <value>HADOOP_MAPRED_HOME=\$HADOOP_HOME/home/hadoop/hadoop/bin/hadoop</value>
</property>
<property>
    <name>mapreduce.map.env</name>
    <value>HADOOP_MAPRED_HOME=\$HADOOP_HOME/home/hadoop/hadoop/bin/hadoop</value>
</property>
<property>
    <name>mapreduce.reduce.env</name>
    <value>HADOOP_MAPRED_HOME=\$HADOOP_HOME/home/hadoop/hadoop/bin/hadoop</value>
</property>
</configuration>

Lastly, tweak the yarn-site.xml to define the auxiliary services for NodeManager.

nano $HADOOP HOME/etc/hadoop/yarn-site.XML

Copy the below code.

<configuration>
<property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
</property>
</configuration>

Step 12: Start the Hadoop Cluster

Before starting Hadoop, format the namenode directory:

hdfs namenode -format

With the Namenode formatted, kick off your Hadoop cluster using the start-all.sh script.:

start-all.sh

Check all services are running with:

jps

Step 13: Accessing Hadoop's Web Interfaces

If necessary, install net-tools to enable the use of the ifconfig command, which helps to retrieve your IP address. Access the Namenode and Resource Manager by entering your server's IP address followed by the appropriate port numbers in a web browser.

sudo apt install net-tools

Step 14: Stopping Hadoop Services

When needed, you can stop all running Hadoop services with the stop-all.sh script.

sudo apt install net-tools
